1. 二分法

排序数组的查找问题首先考虑使用二分法解决，可将时间复杂度降低到对数级别。

  • 为什么当 pivot 在右排序数组时让 high = pivot，左排序数组时让 low = pivot + 1(而不是 low = pivot)

    因为旋转点是”右排序数组的首个元素”，如果 pivot 在右排序数组，则旋转点位于 [low, pivot]；如果 pivot 在左排序数组，则旋转点只能位于（pivot, high]。

    考虑输入 list = [1, 0]，low = 0, high = 1, pivot = 0, numbers[pivot] > numbers[high]，说明 pivot 在左排序数组中

    • 假如令 low = pivot = 0， 程序将陷入循环;
    • 若令 low = pivot + 1 = 1， 将取得正确旋转点 numbers[1] = 0。

    说明 low = pivot 的操作将导致 low = pivot 时陷入循环。

  • 为什么选择 pivot 与 high 比较而不是 low 比较

二分查找的目的是通过确定 pivot 在左排序还是右排序数组来缩小查找区间，而 pivot 和 low 比较存在排序数组判断错误的可能。

考虑输入 list 为完全升序数组 [1, 2, 3, 4], numbers[pivot] > numbers[low]，判断 pivot 在左排序数组，然而该 list 只有右排序数组。
